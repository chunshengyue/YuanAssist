功能概览

YuanAssist能做什么？。

01

跟打模式

把回合动作整理成表格后交给悬浮窗执行，表格形式不限（文字/图片/辟雍学府json），支持多种导入方式。

02

录制模式

直接操作就可以同时记录，录制完成的脚本可以保存、导入和复用，适合生成表格以及洞窟重复操作。

03

攻略表格导入

可以一键导入任何形式的攻略表格图片，OCR转化为可运行脚本。

04

日常自动化

围绕高频任务做模块化入口，例如刷鸟食、刷624和其他重复操作。

05

攻略社区

攻略板块有其他广陵王上传的攻略，支持一键导入。

06

星石拼图

针对星石背包场景，可以进行拼图获取完整星石截图。并且可以一键统计星石数量，减少整理信息的时间成本。

07

轻用户模式

YuanAssist支持识别设备ID一键注册，没有繁琐的步骤。注册后支持发布自己的攻略和脚本，成为创作者。

08

后续可扩展

在目前图像识别和无障碍模式的支持下，YuanAssist未来可以为更多任务提供自动化支持。

上手教程

YuanAssist是低门槛，易操作的APP，这里给出详细教程。

教程 01

悬浮窗是YuanAssist的核心，这一步很简单，手机会弹出提示

  1. 打开应用首页，运行想要的功能，分为战斗版悬浮窗和日常版悬浮窗。
  2. 开启悬浮窗权限，只需要在第一次运行的时候授予。
  3. 开启无障碍权限，这是安卓端自动化的核心权限，每次运行都要重新授予。
  4. 回到主页，悬浮窗会自动弹出。
教程 02

跟打模式是YuanAssist的核心功能，告别手动凹关

  1. 打开战斗版悬浮窗，切换到跟打模式。
  2. 点击导入按钮。
  3. 导入方式有文本、图片、录制模式录制的表格，以及保存的脚本。
  4. 导入之后可以在设置修改战斗参数，这里的操作延时保守可以设置的长一点。
  5. 点击开始跟打，会自动最小化，小窗上有停止按钮、暂停按钮，以及倒计时。
  6. 表格运行结束后，会自动停止。
教程 03

录制自己的表格，让一次操作变成可复用脚本

  1. 打开战斗版悬浮窗，在画面稳定后点击开始录制。
  2. 按照正常打法完成操作，系统会自动记录回合与动作顺序。
  3. 需要手动切换至下一回合。
  4. 录制结束后点击停止按钮。
  5. 保存为脚本或表格，后续可以继续编辑、导出和分享。
教程 04

重复任务的自动化，是时代所趋

  1. 在主页点击日常版的设置按钮。
  2. 选择你想要进行的日常任务。
  3. 修改设置，如刷鸟食需要设置四个任务做哪几个，以及运行结束方式。
  4. 点击导入，切换到游戏对应页面点击开始。
  5. 刷鸟食运行位置：除了主界面右边（带有观星入口）以外都可，会自动导航。
  6. 刷624运行位置：除了主界面右边（带有观星入口）以外都可，会自动导航。
  7. 星石拼图运行位置：星石背包滑到最上方，点击开始。
  8. 日常任务涉及到图像识别，不同手机差异巨大，遇到问题请移步[答疑]部分
教程 05

有没有更快的导入方法？有！

  1. 点击主页第三栏，攻略
  2. 寻找或者搜索你想要跟打的关卡，页面上会显示阵容
  3. 点击之后，可以确认命盘、练度等是否符合
  4. 最下方可以点击导入，导入的时候，所有参数、指令都会一起导入，无需任何调整
  5. 开始战斗！！
  6. 发布攻略：在攻略板块点击＋号，可以发布攻略（必须一键注册之后）
  7. 查看/修改发布：在主页-我的，可以查看自己发布以及收藏的攻略

常见问题

最简单的解决方法：
小红书搜索春笙月，以评论 / 私信的方式询问，页面底部有小红书主页链接。

Q

如何更新版本？

A：版本有更新时，点进软件会有公告。点击主页检查更新，可选应用内更新/浏览器更新。

Q

无法设置无障碍权限？

A：鸿蒙部分版本无法启用无障碍权限，请见谅。

Q

跟打的时候，漏掉某个角色的操作？

A：一种可能是延时设置的短了，部分机型存在小概率（约十分之一）点击无效的情况。如无法解决，请联系作者进行义诊。

Q

刷鸟食、刷624，中间停止了？

A：1.可能是网络卡顿，导致操作游戏画面未及时切换。2.可能是某张素材在您的手机上不太适配，替换方法见下问。3.可能是您的手机运行时有卡顿感，刷鸟食和刷624设置界面都有低配机型适应，在网络/设备不良的情况下，可以试试改成500ms。

Q

无法识别某个图片怎么办？

A：主页-功能测试-上传停止位置的截图-选择对应的文件-点击一键替换，就可以使用自己截图的素材了，百分百能匹配到。若是对应的文件不清楚，请私信询问。

Q

私信询问的时候，需要注意什么？

A：如果停止的那一次运行日志有内容，请带上运行日志最后的片段截图。

Q

无障碍模式有没有风险？

A：有，但是YuanAssist原码已上传github，只使用无障碍权限的点击操作，本质上是自动化工具，不修改游戏数据，和maayuan原理一致。

Q

有没有上线iPhone系统的风险？

A：没有，因为iPhone没有无障碍权限。

Q

能不能在模拟器上运行？

A：理论上可以，但是我试了mumu模拟器无法获取屏幕截图，其他模拟器或许可行，另外模拟器建议使用maayuan。
